<section name="8c19" class="section section--body section--first section--last"><div class="section-divider"><hr class="section-divider"></div><div class="section-content"><div class="section-inner sectionLayout--insetColumn"><h3 name="9fc5" id="9fc5" class="graf graf--h3 graf--leading graf--title">Code Signing 流程與名詞解釋</h3><p name="d956" id="d956" class="graf graf--p graf-after--h3">這次的筆記重點在於 Code-Signing 的流程，以及各項的名詞解釋，針對這一段，每次念完要用的時候都會忘記，所以乾脆自己寫一下筆記，順便也幫助迷途的開發者。</p><p name="3e83" id="3e83" class="graf graf--p graf-after--p">整段的過程大致上分為下列幾個步驟：</p><ol class="postList"><li name="ab34" id="ab34" class="graf graf--li graf-after--p">從本地 KeyChain 產生 .codeSigningRequest</li><li name="08e0" id="08e0" class="graf graf--li graf-after--li">上傳到 Apple Devleoper 進行加簽，並且下載加簽完成的 .certificate</li><li name="82c4" id="82c4" class="graf graf--li graf-after--li">安裝 .certificate 到本地的 KeyChain 當中</li><li name="fbd0" id="fbd0" class="graf graf--li graf-after--li">從 Apple Developer Panel 註冊 App Bundle Identifier</li><li name="4681" id="4681" class="graf graf--li graf-after--li">下載 Provisioning Profile 到本地端的 XCode 進行匯入</li></ol><p name="6349" id="6349" class="graf graf--p graf-after--li">上述有幾個重點可以做一下註記</p><h3 name="6e4a" id="6e4a" class="graf graf--h3 graf-after--p">code signing process</h3><p name="a39e" id="a39e" class="graf graf--p graf-after--h3">之所以需要透過 codeSigningRequest 上傳到 Apple Devleoper 進行加簽的原因，是因為需要驗證開發者，確保這個 app 是由這台電腦所進行開發，有點像是簽名畫押的感覺，然後因為我們在安裝 XCode 的時候，就會一並把根憑證也安裝進去，所以透過 Apple 所發出的憑證，電腦自然就會信任了。</p><h3 name="982f" id="982f" class="graf graf--h3 graf-after--p">為什麼需要第三方來將公鑰加簽成憑證呢？</h3><p name="a97f" id="a97f" class="graf graf--p graf-after--h3">因為全世界能解開我用這把私密金鑰加密的訊息只有和它成對的公開金鑰，如果能確定這把公開金鑰是我發行的，你就能用這把公開金鑰來驗證我的身分，所以公開金鑰需要有第三方公正機構加簽背書。</p><p name="99d6" id="99d6" class="graf graf--p graf-after--p">所以其實簡單來說，憑證 ＝ 公鑰 ＋ 數字簽名 ＋ 使用期限，然後透過信任的第三方機構來做發行，確保此憑證的合法性。</p><h3 name="800f" id="800f" class="graf graf--h3 graf-after--p">憑證的種類</h3><p name="2dfe" id="2dfe" class="graf graf--p graf-after--h3"><strong class="markup--strong markup--p-strong">Development Certificate</strong></p><ul class="postList"><li name="d53b" id="d53b" class="graf graf--li graf-after--p">iOS App Development<br>簽署測試用的APP以安裝在實機上。</li><li name="682e" id="682e" class="graf graf--li graf-after--li">Apple Push Notification service SSL (Sandbox)<br>當你的APP需要使用到Apple Push Notification功能時需要使用此種憑證，開啟沙箱功能。</li></ul><p name="7e84" id="7e84" class="graf graf--p graf-after--li"><strong class="markup--strong markup--p-strong">Production Certificate</strong><br>Production 憑證又分成5種各針對不同需求的 APP</p><ul class="postList"><li name="ef63" id="ef63" class="graf graf--li graf-after--p">App Store and Ad Hoc<br>使用於提交到 App Store 或特殊用途的簽證如：TestFlight。</li><li name="f564" id="f564" class="graf graf--li graf-after--li">Apple Push Notification service SSL (Production)<br>當提交的應用程式需要使用 Apple Push Notification 功能的時候。</li><li name="a68a" id="a68a" class="graf graf--li graf-after--li">Pass Type ID Certificate<br>當APP需要更新Passbook資料的時候</li><li name="96d4" id="96d4" class="graf graf--li graf-after--li">Website Push ID Certificate</li><li name="c017" id="c017" class="graf graf--li graf-after--li">VOIP Services Certificate</li></ul><h3 name="611c" id="611c" class="graf graf--h3 graf-after--li">bundle identifier</h3><p name="306f" id="306f" class="graf graf--p graf-after--h3">app 的識別碼，可以將其想成是 app 的身分證字號，對每一個 app 來說，這組識別碼是獨一無二的。<br>主要分成兩種</p><ol class="postList"><li name="e8d8" id="e8d8" class="graf graf--li graf-after--p"><strong class="markup--strong markup--li-strong">Explicit</strong>: 用來針對單一 app 命名，通常是透過反網址的方式進行。例如 com.nicklee.MyApp 如果需要使用 Push Service 則一定得使用該種命名方式。</li><li name="23ce" id="23ce" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Wildcard</strong>: 透過匹配符號的概念，或是取得部份符合命名規則的 app ，例如 com.nicklee.* 這個的意思是指開頭是 com.nicklee 底下的全部 app，主要是用來進行相關 app service 的授權。</li></ol><h3 name="6585" id="6585" class="graf graf--h3 graf-after--li">provisioning profile</h3><p name="2e6c" id="2e6c" class="graf graf--p graf-after--h3">可以想成是包含了 App Id, Certificates, Device 資訊列表的的一個檔案。會被打包進 app 當中，主要是用來確認該 app 是否合法，透過 Certificate 來驗證開發者是誰，然後透過 App Id 來設定啟用相關功能，最後在測試的時候，確定該裝置的 uuid 是否也一並被打包進 device 列表當中。</p><h3 name="ce9d" id="ce9d" class="graf graf--h3 graf-after--p">Reference</h3><p name="6fd8" id="6fd8" class="graf graf--p graf-after--h3 graf--trailing"><a
